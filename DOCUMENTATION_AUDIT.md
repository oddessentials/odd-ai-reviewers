## EXECUTIVE SUMMARY

The odd-ai-reviewers repository has **9,775 lines of documentation** distributed across:

- **11 root-level .md files** (mostly working files, not public docs)
- **~45 docs/ markdown files** organized into 8 logical sections

### Critical Findings

1. **Root-level file sprawl**: 5 analysis files (`RESEARCH.md`, `PROMPT_ANALYSIS_FINDINGS.md`, `PROMPT_MODIFICATION_MAP.md`, `FINDING_LIFECYCLE_ANALYSIS.md`, `BENCHMARK_RECORD_OPTIONS_ANALYSIS.md`) are **developer-internal** and should move to `docs/analysis/` folder.

2. **Redundancy in setup guides**: GitHub, Azure DevOps, and OSCR platform guides have **40-50% overlapping content** (workflow structure, configuration, secrets). Opportunity to extract common patterns.

3. **Audience gaps**:
   - **Azure DevOps users**: Permissions guide exists but scattered; no end-to-end enterprise example
   - **CLI users**: Local review works but lacks "why use local vs CI" guidance
   - **GitHub Enterprise users**: Free-tier examples exist but enterprise-specific permissions missing

4. **Stale reference**: `docs/local-review.md` exists at root level AND documented in CLI reference — single source of truth unclear.

5. **Inconsistent indexing**: Some sections have `index.md` (getting-started, examples, configuration, reference) but not all. Manifest generation relies on explicit file patterns.

```markdown
# GitHub Permissions Checklist

## For Repository Admins

- [ ] Add `.github/workflows/ai-review.yml` workflow file
- [ ] Add repository secrets:
  - [ ] `ANTHROPIC_API_KEY` (or `OPENAI_API_KEY`)
  - [ ] Optional: `OLLAMA_BASE_URL` (local LLM)
- [ ] Enable workflow permissions:
  - Go to Settings → Actions → General → Workflow permissions
  - Select: "Read and write permissions"
  - Check: "Allow GitHub Actions to create and approve pull requests"
- [ ] Enable required status check (optional, for enforcement):
  - Go to Settings → Branches → Add rule for `main` or target branch
  - Require "AI Review" status check to pass

## For Organization Admins (Enterprise)

- [ ] Approve GitHub App usage (if using first-party integration)
- [ ] Configure IP allowlist (if applicable)
- [ ] Set up secrets at organization level (for multi-repo use):
  - Go to Settings → Secrets and variables → Actions → Organization secrets
  - Add `ANTHROPIC_API_KEY` with environment restriction (optional)
```

#### 3d. Create "Troubleshooting: Common Errors"

**New file:** `docs/getting-started/troubleshooting.md`

**Content:**

```markdown
# Troubleshooting Guide

## GitHub Issues

### "404 Model not found" Error

**Cause:** Model-provider mismatch. API key doesn't support the model.

**Solution:** See [Provider Selection](../configuration/provider-selection.md#model-matching)

### Workflow Doesn't Run on Fork PRs

**Expected behavior:** Reviews don't run on fork PRs by default (security).

**Solution:** Approve the fork PR, or set `trusted_only: false` in `.ai-review.yml` to allow forks.

## Azure DevOps Issues

### "Invalid Authorization" on PR Comments

**Cause:** `System.AccessToken` not passed to pipeline or scopes too restricted.

**Solution:** See [ADO Permissions Checklist](../platforms/azure-devops/permissions-checklist.md#enable-systemaccesstoken)

## Local Review Issues

### "Semgrep not found" Error

**Cause:** Semgrep not installed, but configured as a static agent.

**Solution:** Install semgrep: `pip install semgrep`

See [Local Review](./local-review.md#prerequisites) for full setup.
```
